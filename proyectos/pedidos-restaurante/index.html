Comandas de cocina en tiempo real.

Aplicación web online desarrollada en PHP utilizando el framework Laravel para la gestión de pedidos a cocina de un restaurante en tiempo real (websockets Pusher.com). Permite, desde un panel web, la comunicación en tiempo real entre cocineros y camareros sobre estado comandas. Características principales:

  1. Sistema de registro e identificación de trabajadores del restaurante.
  2. Tres roles operativos:
  • Administador: control total sobre cuentas de trabadores registrados, estado de cuenta activa/desactivada y tipo cuenta (administrador, camarero, cocinero).
  • Camarero: operativa sobre productos y stock, toma de comandas y cambio de estados “Anotar pedido->Pendiente” o “Preparado->Entregado”.
  • Cocinero: operativa sobre cambio de estado de pedido de Pendiente->Preparado y cambio de estado de producto a “no stock”.
  1. Sistema de notificaciones en tiempo real utilizando websockets mediante ventanas emergentes para avisar sobre nuevos pedidos a cocineros o avisar a camareros cuando el pedido está listo en cocina.

English:

Online web application developed in PHP using Laravel framework for managing real-time kitchen orders in a restaurant. It allows, through a web panel, real-time communication between chefs and waiters about the status of orders. Main features:

  1. System of registration and identification of restaurant workers.
  2. Three operational roles:
  • Administrator: absolute control over registered worker accounts, active / deactivated account status and account type (administrator, waiter, chef).
  • Waiter: operation on products and stock, taking orders and change of status “Annotate order -> Pending” or “Ready -> Delivered”.
  • Chef: operation on change of order status Pending -> Ready and change of product status to “no stock”.
  1. Real-time notification system using websockets through pop-up windows to notify chefs about new orders or notify waiters when the order is ready in the kitchen.
